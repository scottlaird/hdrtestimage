# hdrtestimage

An image for testing HDR, along with Go code for generating it.

This bit of Go code generates a 16-bit TIFF file that contains a test
image that covers a very wide range of brightness and color values,
and is intended to be used for testing HDR image conversion pipelines.

This image is currently written out as a TIFF file with no color
profile attached due to limitations in Go's image writing code.  It is
intended to be Rec. 2100 PQ, and I've included an AVIF file generated
via Photoshop using the TIFF as a source and Hasselblad's Rec.2100 PQ
color profile as a colorspace.  I'll replace this with a
correctly-generated file once I find a tool that can do it in Go.

![test image](hdrtestimage.avif)

## Notes on the image

This image has 2 main components, grayscale bars on the left and right
(inverted from each other) and a colorwheel in the middle.

### Gray luminance bars

The gray bars *should* run from full white down to 16 stops below full
white.  The color levels were generated by using the [PQ transfer
function](https://en.wikipedia.org/wiki/Perceptual_quantizer) to
transform a brightness target in nits into a pixel value.  The bars
start with a maximum of 10,000 nits and then divide the number of nits
in half for each stop, ending around 0.15 nits.

Your display is not expected to be able to display the full range.

The gray bars include a strip on the left and right that are offset by
1 stop.  This makes it clear when clipping is occuring, because the
middle bar and the left bar will be the same color.

### Color wheel

The color wheel includes concentric rings of the
[OKLCH](https://en.wikipedia.org/wiki/Oklab_color_space) hue spectrum
at various luminance values, from 1 stop to 13 stops down from full
brightness.  The luminance of the `1` color ring should roughly match
the luminance of the `1` line on the gray chart, although I'm not
entirely sure that I'm using the Rec.2020 color primaries for OKLCH so
it's possible that everything is slightly off.
